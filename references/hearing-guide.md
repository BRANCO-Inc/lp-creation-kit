# ヒアリングガイド（3段階）

LPを書く前に、この順で材料を集める。各段階は「①ユーザーへの質問」と「②一次情報ファイルの依頼」の2系統をセットで出す。
質問は一度に全部投げず、段階ごとに区切る。ユーザーは音声入力（AquaVoice等）で思いつくまま答える前提。フィラー・言い直し・重複は無視して意図を抽出する。

一次情報の読み方:
- ローカルファイル → Read
- URL → int-web-fetch（Task経由のサブエージェント。複数URLは並列）
- Google Drive → int-google-drive-manage

---

## STEP 1. ターゲットのヒアリング

### ① ユーザーへの質問（そのまま投げる）

```
【STEP1/3 ターゲット】
このウェビナーを「誰に届けたいか」を教えてください。
正確じゃなくて大丈夫です。AquaVoiceなどで、思いつくまま話してください。

- どんな人ですか？（年代・性別・職業・生活・今やっていること）
- その人は今、何に悩んでいますか？ 何にモヤモヤしていますか？
- 過去に何を試して、何で挫折・失敗してきましたか？
- このままだと、その人が避けたい未来は何ですか？
- 逆に、本当はどうなりたいと思っていますか？
- その人が普段つぶやきそうな「心の声」をそのまま教えてください
```

### ② 一次情報ファイルの依頼

```
ターゲットに関する一次情報があれば、ファイルのパスかURLをください（何個でも）。
例: N1インタビュー、アンケート結果、既存顧客の声、ペルソナ資料、過去LPの反応、Lステップの友だち属性 など
無ければ「なし」でOKです。
```

渡されたら全て読み、ターゲットの「実際の言葉」を抜き出してブリーフに使う。一次情報の生の言い回しは、悩みブロックと受講生の声の精度を大きく上げる。

## STEP 2. コンセプトのヒアリング

### ① ユーザーへの質問

```
【STEP2/3 コンセプト】
ウェビナー（勉強会）のコンセプトを、思いつくまま話してください。

- 一言で言うと、何のウェビナーですか？
- ウェビナーのタイトル（正式 or 仮）は？
- 中身はどんな内容ですか？ 参加すると何がわかる/できるようになりますか？
- 「それができる理由」は？（なぜあなたが教えられるのか / なぜ今なのか / なぜこの方法なのか）
- 参加者が「何を信じたら」申し込むと思いますか？
- 既存のやり方・他の選択肢と、何が違いますか？
- LPに絶対入れたいメッセージ・言葉があれば、それも全部
```

### ② 一次情報ファイルの依頼

```
コンセプトに関する一次情報があれば、パスかURLをください。
例: ウェビナー資料・スライド、台本、商品設計メモ、過去セミナーの反応、講師の実績資料、競合LP など
無ければ「なし」でOKです。
```

ウェビナースライドPDFが来たら file-pdf-ocr で読む。競合LPのURLが来たら mkt-lp-analyze の発想で構造を拾う。

## STEP 3. その他LP作成に必要な情報

### ① ユーザーへの質問

```
【STEP3/3 仕上げの材料】
LPに載せる残りの材料を、わかる範囲で全部ください。曖昧でもOKです。

- 特典は何がありますか？（数と、それぞれの中身・タイトル）
- 実績・数値はありますか？（参加者数・満足度・成功率・事例・Before After）
  ※入力にない数字は創作しないので、根拠のあるものだけで大丈夫です
- 講師は誰ですか？（名前・肩書き・信じてもらえる経歴）
- オファーの条件は？（無料/有料・所要時間・開催形式・日程）
- 参加のハードルを下げる条件はありますか？（カメラオフOK・初心者OK・子連れOK・顔出し不要 など）
- 緊急性に使えるものは？（人数制限・締切・満席実績 など。根拠があるものだけ）
- 申し込みの最終行動は？（LINE追加 / LINEで日程確認 / LINEで特典受け取り）
- その他、LPに載せたいことがあれば全部
```

### ② 一次情報ファイルの依頼

```
概要をまとめたMDファイルや資料があれば、パスかURLをください。
（特典一覧、実績まとめ、講師プロフィール、オファー設計メモ など）
無ければ「なし」でOKです。
```

## ヒアリング後のチェック

ブリーフ（SKILL.md STEP1）を埋める前に、最低限これが揃っているか確認する。足りない項目だけを短く追加質問する。

- [ ] ターゲットの「刺さる悩み」が具体的な生活場面の言葉になっている（一般論ではない）
- [ ] ウェビナータイトルが（仮でも）ある
- [ ] コンセプトの「できる理由」がある
- [ ] CTAの最終行動が1つに決まっている
- [ ] 実績・数値は根拠のあるものだけ（創作していない）

揃ったら、SKILL.md STEP1 のブリーフに圧縮して STEP2（3型並列生成）へ進む。
