# ヒアリングガイド（3段階）

LPを書く前に、この順で材料を集める。各段階は「①ユーザーへの質問」と「②一次情報ファイルの依頼」の2系統をセットで出す。
質問は一度に全部投げず、段階ごとに区切る。ユーザーは思いつくまま話す前提で、フィラー・言い直し・重複は無視して意図を抽出する。

一次情報の読み方（渡されたものに応じて読む）:
- ローカルファイルのパス → そのファイルを読む
- URL → そのページを読む（複数URLは並行して読む）
- クラウド上のファイル → 該当ファイルを読む

---

## STEP 1. ターゲットのヒアリング

### ① ユーザーへの質問（そのまま投げる）

```
【STEP1/3 ターゲット】
このオファーを「誰に届けたいか」を教えてください。
正確じゃなくて大丈夫です。思いつくまま話してください。

- どんな人ですか？（年代・性別・職業・生活・今やっていること）
- その人は今、何に悩んでいますか？ 何にモヤモヤしていますか？
- 過去に何を試して、何で挫折・失敗してきましたか？
- このままだと、その人が避けたい未来は何ですか？
- 逆に、本当はどうなりたいと思っていますか？
- その人が普段つぶやきそうな「心の声」をそのまま教えてください
```

### ② 一次情報ファイルの依頼

```
ターゲットに関する一次情報があれば、ファイルのパスかURLをください（何個でも）。
例: N1インタビュー、アンケート結果、既存顧客の声、ペルソナ資料、過去LPの反応、登録者の属性データ など
無ければ「なし」でOKです。
```

渡されたら全て読み、ターゲットの「実際の言葉」を抜き出してブリーフに使う。一次情報の生の言い回しは、悩みブロックと利用者の声の精度を大きく上げる。

---

## STEP 2. コンセプトのヒアリング

### ① ユーザーへの質問

```
【STEP2/3 コンセプト】
オファーのコンセプトを、思いつくまま話してください。

- 一言で言うと、何のオファーですか？
- オファーのタイトル（正式 or 仮）は？
- 中身はどんな内容ですか？ 受け取ると何がわかる/できるようになりますか？
- 「それができる理由」は？（なぜあなたが提供できるのか / なぜ今なのか / なぜこの方法なのか）
- 受け取る人が「何を信じたら」申し込むと思いますか？
- 既存のやり方・他の選択肢と、何が違いますか？
- LPに絶対入れたいメッセージ・言葉があれば、それも全部
```

### ② 一次情報ファイルの依頼

```
コンセプトに関する一次情報があれば、パスかURLをください。
例: 提供内容の資料・スライド、台本、商品設計メモ、過去の反応、提供者の実績資料、競合LP など
無ければ「なし」でOKです。
```

資料のファイルやスライドが来たら読んで中身を拾う。競合LPのURLが来たら、その構造（どの順で何を見せているか）を観察して参考にする。

---

## STEP 3. その他LP作成に必要な情報

### ① ユーザーへの質問

```
【STEP3/3 仕上げの材料】
LPに載せる残りの材料を、わかる範囲で全部ください。曖昧でもOKです。

- 特典は何がありますか？（数と、それぞれの中身・タイトル）
- 実績・数値はありますか？（利用者数・満足度・成功率・事例・Before After）
  ※入力にない数字は創作しないので、根拠のあるものだけで大丈夫です
- 提供者は誰ですか？（名前・肩書き・信じてもらえる経歴）
- オファーの条件は？（無料/有料・所要時間・提供形式・日程）
- 申し込みのハードルを下げる条件はありますか？（顔出し不要・初心者OK・子連れOK・勧誘なし など）
- 緊急性に使えるものは？（人数制限・締切・満席実績 など。根拠があるものだけ）
- 申し込みの最終行動は？（LINE登録 / 予約 / 特典受け取り など、1つに決める）
- その他、LPに載せたいことがあれば全部
```

### ② 一次情報ファイルの依頼

```
概要をまとめたファイルや資料があれば、パスかURLをください。
（特典一覧、実績まとめ、提供者プロフィール、オファー設計メモ など）
無ければ「なし」でOKです。
```

---

## ヒアリング後のチェック

ブリーフを埋める前に、最低限これが揃っているか確認する。足りない項目だけを短く追加質問する。

- [ ] ターゲットの「刺さる悩み」が具体的な生活場面の言葉になっている（一般論ではない）
- [ ] オファーのタイトルが（仮でも）ある
- [ ] コンセプトの「できる理由」がある
- [ ] CTAの最終行動が1つに決まっている
- [ ] 実績・数値は根拠のあるものだけ（創作していない）

揃ったら、ブリーフに圧縮して3型の並列生成へ進む。
