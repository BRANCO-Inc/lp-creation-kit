# LINE誘導LP構造リファレンス

実LP8本（meta→LINE 無料ウェビナー）の分析から作った、セクション型・役割・文字量・CTA設計・3型の構成。

## 目次

- 標準セクションカタログ
- セクション別ルール（役割・文字量）
- CTA設計
- 3型の構成（型A/型B/型C）

## 標準セクションカタログ

実LPに共通して出てくるセクション。型ごとに順番と比重を変える。

| セクション名 | 役割 |
|---|---|
| `FV` | 誰が・何を・どれくらい簡単に得られるかを一画面で。型ごとにフックを変える |
| `実績バッジ` | 満足度・参加者数・権威をFV直下に数値で（根拠がある時だけ） |
| `CTA` | LINE追加の行動を明示。最低3回、通常3〜5回 |
| `悩み` | ターゲットの心の声を3〜4タイプの吹き出しで当てる |
| `問題提起` | 現状維持の損失・なぜ今までうまくいかなかったか |
| `型反転` | 「才能や努力不足ではなく、◯◯の型を知らないだけ」で自己否定を解除 |
| `解決策` | 今回のウェビナーが解決する範囲を示す |
| `ウェビナー内容` | 学べること・できるようになることを4〜6項目で具体化 |
| `特典` | ◯大特典。数・ラベル・無料を見せる |
| `実績/社会的証明` | 数値・事例・受講生の声 |
| `受講生の声` | リアルな一人称＋属性＋具体的な変化 |
| `講師/商品説明` | 信じられる理由を短く（型Cでは厚く） |
| `安心条件` | カメラオフOK・初心者OK・子連れOK・勧誘なし |
| `LINE受け取りSTEP` | 3STEPで登録後の流れ |
| `FAQ` | 不安だけを消す。3〜5問 |
| `クロージング` | 緊急性・無料性・未来像・二者択一で締めてCTAへ |

## セクション別ルール

| セクション | 文字量目安 |
|---|---|
| FV | メイン16〜34字（2〜4行に分割）。サブ40〜90字。バッジ3個まで |
| 実績バッジ | 数値2〜4個。各「数字＋単位＋ラベル」 |
| CTA | ボタン12〜26字。直前文15〜40字。注記15〜45字 |
| 悩み | 吹き出し3〜4個、各18〜45字。同じ感情を繰り返さない |
| 問題提起 | 80〜180字。データがなければ断定しすぎない |
| 型反転 | 60〜140字。1文で言い切る |
| 解決策 | 120〜260字 |
| ウェビナー内容 | 4〜6項目。H3 8〜22字、本文20〜50字 |
| 特典 | 数を明示。各特典名＋網羅性の一言（「全30項目チェック付き」等） |
| 受講生の声 | 1人60〜160字。属性（年代・職業・家族構成）必須 |
| 講師 | 80〜180字（型Cは240字まで） |
| 安心条件 | 箇条書き3〜5個、各8〜30字 |
| FAQ | 3〜5問。回答は1文 |
| クロージング | 100〜220字 |

## CTA設計

LINE誘導LPでは、フォーム入力の説明をしない。読者が押した後の状態を見せる。

- CTAは最低3回、通常3〜5回。配置はFV直下・内容/ベネフィット後・実績/FAQ後・クロージング
- ボタン文言は毎回大きく変えず、「媒体（LINE）＋無料性＋行動」を1つにまとめる
- 各CTAの直前に短い橋渡し文、各CTAの下に注記を置く

実データのCTAパターン（`japanese-expression-rules.md` の使える型も参照）:

```
ボタン: 今すぐLINE追加して 勉強会に無料参加する
直前文: まずは気軽にLINE登録。勉強会は参加費0円です。
注記: ※LINEが開かない場合、無料勉強会は終了しています。
```

希少性は根拠がある時だけ。満席実績・残席・締切は入力にある場合のみ使う。

## 3型の構成

同じブリーフでも、入口（フック）と前半の順番を変えて3本書く。後半（特典・実績・講師・FAQ・クロージング）はどの型も共通要素を持つが、比重が違う。

### 型A: 特典ご利益先出し型（Offer-First）

刺さる層: 行動ハードルが主な障壁の中〜低関心層。「無料でこんなに貰える」のお得感で手を止めさせる。

```
FV（理想の変化 ＋ 無料/◯大特典バッジ ＋ 実績数値）
→ CTA①
→ 特典の物量見せ（◯大特典をグリッドで。各特典名＋網羅性）
→ ウェビナー内容（得られること3〜4個）
→ CTA②
→ 悩み（軽め。3個）
→ 実績/受講生の声
→ 講師
→ 安心条件
→ FAQ
→ クロージング（特典再掲 ＋ 希少性）
→ CTA最終
```

### 型B: 悩み共感・型反転型（Empathy → Reframe / PASONA）

刺さる層: 問題は自覚しているが行動できない・挫折経験あり層。「自分のことだ」と思わせてから「あなたのせいじゃない」で反転。

```
FV（悩みの代弁 or 「◯◯ってなんか難しそう…」 ＋ 小さな希望）
→ 悩み（吹き出し3〜4連）
→ 問題提起（このまま様子見を続けるとどうなるか）
→ 型反転（「才能や努力不足ではなく、◯◯の型を知らないだけ」）
→ CTA①
→ 解決策（だからこのウェビナーが要る）
→ ウェビナー内容（「9割が間違える◯◯」系で具体化）
→ 特典
→ 受講生の声（同じ悩みからの変化）
→ 講師
→ 安心条件
→ FAQ
→ クロージング（余韻型。「そんなあなたに、このウェビナー」）
→ CTA最終
```

### 型C: 逆転ストーリー・権威先出し型（Story / Authority-First）

刺さる層: 憧れ駆動・半信半疑層。強烈なギャップ・物語・権威で先に心を掴み、「この人から学びたい/これは本物だ」と思わせる。

```
FV（逆転実績/ギャップ or 時代転換の否定フック ＋ 強い数値）
→ ストーリー（Before悲惨 → 転機 → After）or 権威の証明
→ 型反転（「あなたにもできる。才能じゃない」）
→ CTA①
→ 何が違うのか（既存手法との差別化・「業界人が言わない裏話」系）
→ ウェビナー内容
→ 特典
→ 実績/受講生の声
→ 講師プロフィール（権威を厚く）
→ 安心条件（プリバンキング:「怪しいと思う人こそ」）
→ FAQ
→ クロージング（二者択一 or 余韻）
→ CTA最終
```
