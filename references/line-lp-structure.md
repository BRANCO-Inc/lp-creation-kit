# 誘導LP構造リファレンス

無料オファー・体験・予約・診断・登録などへ誘導するLP本文の、標準セクション・役割・文字量・CTA設計・3型の構成。あらゆるジャンル（教育/美容/金融/医療/法律/BtoBサービス等）で共通に使える。商品本体の直販ではなく、CTAは登録/予約への一歩に置く。

## 目次

- 標準セクションカタログ
- セクション別ルール（役割・文字量）
- CTA設計
- 3型の構成（型1 オファー先出し / 型2 自己投影・反転 / 型3 権威・実証先出し）
- 横断の緊張装置

## 標準セクションカタログ

誘導LPで繰り返し使うセクション。型ごとに順番と比重を変えて組み替える。画像LPに載る粒度で考える（1セクション＝1スクロール前後）。

| セクション名 | 役割 |
|---|---|
| `FV` | 誰が・何を・どれくらい手軽に得られるかをファーストビュー1画面で。型ごとにフックを変える |
| `実績バッジ` | 満足度・利用者数・第三者評価をFV周辺に数値で（根拠がある時だけ） |
| `CTA` | 登録/予約への行動を明示。最低3回、通常3〜5回 |
| `悩み` | ターゲットの心の声を3〜4タイプの吹き出しで言い当てる |
| `問題提起` | 現状維持の損失・なぜ今までうまくいかなかったか |
| `反転` | 「才能や努力不足ではなく、[ ]の順番／知識が無かっただけ」で自己否定を解除し原因を外部化する |
| `解決策` | このオファーが解決する範囲を示す |
| `中身（提供価値）` | 受け取ると何が分かる・できるようになるかを4〜6項目で具体化 |
| `特典` | [ ]大特典。数・ラベル・無料を見せる |
| `実証（実績・利用者の声）` | 数値・事例・利用者の一人称の声 |
| `提供者プロフィール` | 信じられる理由を短く（型3では厚く） |
| `安心条件` | 初めてでもOK・しつこい勧誘なし・途中離脱OK など不安の打ち消し |
| `受け取りSTEP` | 登録後の流れを3STEPで |
| `FAQ` | 不安だけを消す。3〜5問 |
| `クロージング` | 希少性・無料性・未来像・二者択一で締めてCTAへ |

## セクション別ルール

役割と文字量の目安。あくまで目安で、画像LPの1枚に収まる量に調整する。

| セクション | 役割の要点 | 文字量目安 |
|---|---|---|
| FV | 一画面で「自分ごと」と「得」を立てる | メイン16〜34字（2〜4行に分割）。サブ40〜90字。バッジ3個まで |
| 実績バッジ | 数値で初速の信頼を作る | 数値2〜4個。各「数字＋単位＋ラベル」 |
| CTA | 押した後の状態を見せる | ボタン12〜26字。直前文15〜40字。注記15〜45字 |
| 悩み | 同じ感情を繰り返さず網羅する | 吹き出し3〜4個、各18〜45字 |
| 問題提起 | 損失を可視化する。根拠なく断定しない | 80〜180字 |
| 反転 | 自己否定を1文で解除する | 60〜140字。1文で言い切る |
| 解決策 | このオファーの守備範囲を明示 | 120〜260字 |
| 中身（提供価値） | 抽象でなく「分かる・できる」で書く | 4〜6項目。見出し8〜22字、本文20〜50字 |
| 特典 | 数とラベルと網羅性を見せる | 数を明示。各特典名＋網羅性の一言（「全[ ]項目チェック付き」等） |
| 実証（実績・利用者の声） | 具体行動＋変化＋一人称。創作禁止 | 1人60〜160字。属性（年代・職業・状況）必須 |
| 提供者プロフィール | 信じられる理由を絞る | 80〜180字（型3は240字まで） |
| 安心条件 | 直前の不安だけを消す | 箇条書き3〜5個、各8〜30字 |
| 受け取りSTEP | 登録後の道筋を見せる | 3STEP。各STEP見出し＋1文 |
| FAQ | 申込み直前の不安だけ消す | 3〜5問。回答は1文 |
| クロージング | 温度を上げて最後の一歩を後押し | 100〜220字 |

## CTA設計

誘導LPのCTAは、フォーム入力の手順を説明しない。読者が押した後に手に入る状態を見せる。

- CTAは最低3回、通常3〜5回。配置は `FV直下` / `中身・ベネフィット後` / `実証・FAQ後` / `クロージング`
- ボタン文言は毎回大きく変えず、「媒体＋無料／低リスク＋行動」の3要素を1つにまとめる
  - 例: `[媒体]で受け取って 無料体験に申し込む` / `[媒体]を登録して 無料診断を受け取る` / `予約フォームへ進んで [初回0円]を確保する`
- 各CTAの直前に短い橋渡し文（次の一歩のハードルを下げる1文）を置く
- 各CTAの下に注記（リスク反転・条件・期限切れ時の挙動）を置く

中立例（ジャンルを散らした自作例。実在のコピーではない）:

```
ボタン: [媒体]を登録して 無料カウンセリングに進む
直前文: まずは登録だけ。カウンセリングは費用ゼロ、無理な勧誘もありません。
注記: ※ボタンが開かない場合、無料カウンセリングは募集を終了しています。
```

```
ボタン: [媒体]で 7日間の無料レッスンを受け取る
直前文: 続けられるか不安でも、最初の7日は0円で試せます。
注記: ※先着[ ]名まで。定員に達し次第クローズします。
```

希少性・緊急性は根拠がある時だけ使う。残席・締切・募集終了予告は、入力に実際の数字や期限がある場合のみ書く。創作しない。

## 3型の構成

同じブリーフでも、入口（フック）と前半の順番を変えて3本書き分ける。後半（特典・実証・提供者・安心条件・FAQ・クロージング）はどの型も共通要素を持つが、比重が違う。各型の起点心理と刺さる層を踏まえて並べる。

### 型1 オファー先出し型（報酬を先に見せる / Offer-led）

起点心理は接近欲求＋損得勘定。検討の前に「手に入るもの」を見せ、低摩擦で最初のYESを取る。刺さる層は関心が浅い・冷たい流入で、行動ハードルが最大の障壁。FVで無料・特典の物量・なりたい理想の状態・「できない理由はもう無い」（コスト/スキル/時間/リスクの障壁除去）を見せる。特典・無料はCTA毎に再掲する。

```
FV（報酬＋無料/[ ]大特典バッジ＋実績数値）
→ CTA①
→ 特典/中身（[ ]大特典の物量見せ。各特典名＋網羅性）
→ CTA②
→ 悩み（軽め。3個）
→ 実証（実績・利用者の声）
→ 提供者プロフィール
→ 安心条件
→ FAQ
→ クロージング（報酬再掲＋希少性）
→ CTA最終
```

### 型2 自己投影・反転型（読者自身を鏡に映す / Mirror-led）

起点心理は自己関連付け＋自己効力感の回復。読者の悩み/現状/挫折を一撃で言語化（鏡）→「あなたのせいじゃない。[ ]を知らない/順番が違っただけ」と原因を外部化（免責）→反転。語り手を「元・同じ立場の人」に置く逆転ストーリーもこの型に含む。刺さる層は挫折経験・自己評価が低い・自分で行動を止めている層。FVで悩みの代弁、または「[ ]って、なんだか難しそう」、または元どん底の自分の独白を見せる。警戒の先回り（プリバンキング）を2箇所入れる。

```
FV（悩み/独白）
→ 悩み（吹き出し3〜4連）
→ 問題提起（現状維持を続けるとどうなるか）
→ 反転（免責。「[ ]を知らなかっただけ」）
→ CTA①
→ 解決策（だからこのオファーが要る）
→ 中身（提供価値）
→ 実証（同じ境遇からの声）
→ 提供者プロフィール
→ 安心条件
→ FAQ
→ クロージング（余韻型）
→ CTA最終
```

### 型3 権威・実証先出し型（信頼を先に立てる / Proof-led）

起点心理は権威への信頼転移＋判断の外注。読者が自力で良し悪しを判断できない領域で、実績・権威・第三者の証明・変化の証明（ビフォーアフター）を先に出し「これは本物だ」と信じさせてからベネフィットを乗せる。刺さる層は半信半疑・高単価/専門領域・「騙されたくない」層。FVで圧倒的な実績数値、第三者の評価・掲載、変化の証明、規模のいずれかを見せる。

```
FV（実績/権威＋強い数値）
→ 実証（証明の多層積み上げ）
→ CTA①
→ 解決策（何が違うのか。差別化）
→ 中身（提供価値）
→ 実証（声・数値）
→ 提供者プロフィール（厚め・権威）
→ 安心条件（プリバンキング）
→ FAQ
→ クロージング（二者択一/余韻）
→ CTA最終
```

## 横断の緊張装置

型ではなく、全型で前半・問題提起・クロージングに差し込める点火装置。単独でLP全体を支えないので独立した型にはしない。根拠がある時だけ使う。

- 時代転換/危機（FOMO）: 「[ ]の前提は変わった」「今動かないと差が開く」。問題提起やクロージングで温度を上げる増幅装置。
- 緊急性/希少性: 残席・締切・募集終了予告。CTA直前に置く。実際の数字・期限が入力にある時だけ。

これらは温度を上げるための装置であり、訴求の主軸にはしない。中身・実証・反転が弱いLPに緊張装置だけ盛っても、不信を強めるだけになる。
