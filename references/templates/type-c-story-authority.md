# 型C: 逆転ストーリー・権威先出し型（Story / Authority-First）

## この型の性格

強烈なギャップ・物語・権威で先に心を掴む型。
刺さる層は、憧れ駆動・半信半疑の層。「この人から学びたい」「この方法は本物だ」と先に思わせてから、再現性（あなたにもできる）へ橋渡しする。
FVは、どん底からの逆転実績か、「[既存のやり方]の時代は終わった」という時代転換の否定フックで、一瞬で注意を奪う。

実LPの代表: opt-in（「時給10円やった私が8億」）/ fusion-ai（「ChatGPTの時代、終わりました」）/ zeromin（手出し0円→500万、フジテレビ特集）。

## 書き方の指針

- FVは2択。①逆転実績/ギャップ（どん底→強い数字）か、②時代転換の否定フック（「[既存]は終わった」）。強い数値をFV直下に置く
- ストーリーはBefore（悲惨・具体的）→転機（ある気づき）→After（実績）の順。一人称・口語で人間味を出す
- 権威は具体で（メディア名・登録者数・受賞・実績金額）。抽象的な「すごい人」にしない
- 「あなたにもできる。才能じゃない」で型反転し、憧れを自分ごとに変える
- プリバンキング（「怪しいと思う人こそ」）で警戒を先に解く
- 「新時代の」「市場価値の高い」等の煽り語は使わない。具体の物語と数字で語る（japanese-expression-rules.md 参照）

## セクション構成と雛形

H1はブリーフの[ウェビナータイトル]をそのまま使う（`# LPテキスト`のような汎用見出しにしない）。

ブリーフにBefore側の「どん底の数字」が無い時の作り方:
- 数字を捏造しない。代わりに状態を具体・口語で描く（「ツールを覚えても、何も変わらなかった」「毎晩このままでいいのかと思っていた」）
- After側の根拠ある数字（ブリーフにあるもの）と、Beforeの状態描写の落差でギャップを作る
- それでもストーリー材料が薄い（講師の物語も実績も弱い）なら、型Cは無理に立てず、型A/型Bを推す判断をメインに報告する

## チェック（この型を出す前に）

- [ ] FVが逆転実績か時代転換フックで、一瞬で注意を奪う

- [ ] 権威が具体（メディア・数字・受賞）で、抽象的でない
- [ ] 型反転で「あなたにもできる」へ橋渡しできている
- [ ] プリバンキングで警戒を先に解いている
- [ ] 「新時代の」等の煽り語を使っていない／実績は入力にある根拠のみ
