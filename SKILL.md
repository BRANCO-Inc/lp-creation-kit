# LINE誘導LPライティング

## Overview

meta/SNS広告から無料ウェビナー（勉強会・無料動画・説明会）へLINE登録で誘導するLPの本文Markdownを作る。

このSkillの中核は3つ。

1. **3段階ヒアリング**: ターゲット → コンセプト → その他情報の順で、ユーザーへの質問と一次情報ファイルの2系統で材料を集める
2. **3型を並列生成**: 集めた材料から、性格の異なる3つの型を各1エージェントで同時に書き起こす
3. **99点まで自己監査**: 各原稿を `references/audit-checklist.md` の99点ルーブリックで採点し、未達なら直してから出す

配布用の独立Skillとして動く。特定環境の絶対パスや非公開ナレッジに依存しない。材料はユーザー入力・添付ファイル・このSkill内の `references/` だけで完結させる。

実LP8本（fusion-ai / linkup / realus / opt-in / zeromin 系）の文字起こしを分析して型・表現・監査基準を作った。`references/` の中身はその実データに裏付けされている。

## いつ何を読むか

| タイミング | 読むファイル |
|---|---|
| ヒアリングを始める前 | `references/hearing-guide.md` |
| 型を選ぶ・生成を割り振る前 | `references/templates/` の3ファイル |
| 各エージェントが書く時 | 担当テンプレ + `references/line-lp-structure.md` + `references/japanese-expression-rules.md` |
| 表現に詰まった時の実コピー集 | `references/copy-swipe-file.md`（実LP8本から抽出した原文。丸写しでなく言い回しの参考に） |
| 書いた後 | `references/audit-checklist.md` |

## ワークフロー

### STEP 0. ヒアリング（3段階）

`references/hearing-guide.md` を開き、その順番で進める。要点だけ再掲する。

1. **ターゲット**: ①ユーザーへの質問（悩み・理想を思いつくまま話してもらう）＋②ターゲットの一次情報ファイル（N1・アンケート・顧客の声など）のパス/URL
2. **コンセプト**: ①ユーザーへの質問（一言で言うと/タイトル/内容/できる理由/信じてほしいこと）＋②コンセプトの一次情報ファイル（ウェビナー資料・台本・商品設計メモ）
3. **その他情報**: ①ユーザーへの質問（特典・実績・講師・オファー・CTA・安心条件・緊急性）＋②概要をまとめたMD等のパス/URL

ルール:
- 各段階で「ユーザーへの質問」と「一次情報ファイルの依頼」を必ずセットで出す
- 一次情報が渡されたら読む。ローカルは Read、URLは int-web-fetch、Driveは int-google-drive-manage
- ユーザーが「全部入力済み」「もう材料はある」と言ったら、不足項目だけ確認して先へ進む
- 音声入力で材料が来る前提。フィラー・言い直しは無視して意図を抽出する

### STEP 1. 材料を1枚のブリーフに圧縮する

3段階で集めた材料を、生成エージェントへ渡す共通ブリーフに圧縮する。長い一次情報も必ずこの形へ要約する。

```md
## LPブリーフ
ターゲット:（属性 / 今の行動 / 刺さる悩み / 避けたい未来 / なりたい状態）
刺さる悩みTOP3:（具体的な生活場面の言葉で。3つ）
信じさせたいコンセプト:（一言で / できる理由 / 既存選択肢との違い）
ウェビナータイトル:
オファー:（無料/有料 / 所要時間 / 形式 / 日程）
特典:（数 と 各特典名）
実績・数値:（入力にある根拠のみ。創作禁止）
講師:（名前 / 肩書き / 信じられる理由）
安心条件:（カメラオフOK・初心者OK・子連れOK など）
緊急性:（人数制限・期限。根拠がある時だけ）
CTA最終行動:（LINE追加 / 日程確認 / 特典受け取り）
```

CTA文言やURLが未確定なら、本文では仮置きで `LINEで無料参加枠を受け取る` を使う。

### STEP 2. 3型を並列生成する（3エージェント / 1型1エージェント）

3つの型を、それぞれ別エージェントで**同時に**書く。Task tool（Claude Code）または並列実行で3エージェントをスポーンする。

| エージェント | 担当テンプレ | 性格 |
|---|---|---|
| Agent A | `references/templates/type-a-offer-first.md` | 特典ご利益先出し型 |
| Agent B | `references/templates/type-b-empathy-pasona.md` | 悩み共感・型反転型 |
| Agent C | `references/templates/type-c-story-authority.md` | 逆転ストーリー・権威先出し型 |

各エージェントへ渡す指示（テンプレ。3つ生成して同時スポーン）:

```
LP本文をMarkdownで書くタスクです。

まず以下を順に読む:
1. /Users/yudaiishida/Agents/marketing/skills/mkt-lp-write/references/templates/{担当テンプレ}
2. /Users/yudaiishida/Agents/marketing/skills/mkt-lp-write/references/line-lp-structure.md
3. /Users/yudaiishida/Agents/marketing/skills/mkt-lp-write/references/japanese-expression-rules.md
（表現に詰まったら references/copy-swipe-file.md の実コピーを参考にする。丸写し禁止）

LPブリーフ:
（STEP1のブリーフを貼る）

担当テンプレの型・セクション順・心理に厳密に従ってLP本文Markdownを書く。
- japanese-expression-rules.md の禁止表現はゼロ許容。書き終えたら検索して消す
- 入力にない実績・人数・満足度・No.1は創作しない
- 説明や制作意図は本文に混ぜない。LP本文だけを出力する
- 出力後、references/audit-checklist.md で自己採点し、99点未満なら直してから出す

出力: LP本文Markdown（型名を冒頭にコメントで1行だけ付ける）
```

3型を割り振る理由: 同じ材料でも、お得感で動く層・共感で動く層・憧れと権威で動く層は刺さる入口が違う。3本を並べて一番効く入口を選べるようにする。

### STEP 3. 統合・自己監査・提示

3本が返ってきたら、メインが `references/audit-checklist.md` で各本を再採点する。

- 99点未満の本は、該当エージェントへ差し戻すか、メインが該当箇所だけ直す
- 3本に共通の弱点（同じ禁止表現・同じ根拠なし数字）があれば、テンプレ/ルール側の不足としてメモする
- 3本のうち、ブリーフのターゲットに最も合う1本を「推奨」として明示する

最終出力:

```md
## 推奨: 型B（悩み共感・型反転型）
理由:（ターゲットが「やる気はあるが挫折経験あり」のため、型反転で自己否定を解除する入口が最も効く）

---
（型A 全文）
---
（型B 全文）
---
（型C 全文）
```

ユーザーが「1本でいい」「型Bだけ」と指定したら、その型だけ生成・出力する。

## 出力方針

- 最終出力はLP本文Markdownを主役にする
- セクション名は `references/line-lp-structure.md` の標準セクション名を使う
- デザイナー/実装者がそのまま画像LPに載せられる粒度で書く（H2/H3は短く、1セクション詰め込みすぎない）
