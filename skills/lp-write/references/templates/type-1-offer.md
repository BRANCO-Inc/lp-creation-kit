# 型1 オファー先出し型（報酬を先に見せる / Offer-led）

## この型の性格

最初に「手に入るもの」を見せる型。検討させる前に、無料・特典・なりたい状態・障壁の除去を冒頭で提示し、低摩擦で最初のYESを取りにいく。読者がまだ温まっていない段階でも、「とりあえずもらっておくか」と手を伸ばさせるのが狙い。

起点になる心理は、接近欲求（良いものに近づきたい）と損得勘定（得しそうなら取りに行く）。だから本文の前半は、説得や物語ではなく「何がもらえるか」「いくらかかるか（＝かからない）」「どれだけ楽か」を具体で見せることに使う。

## 刺さる層

- 関心が浅い、まだ温度の低い流入（広告クリック直後、名前を知ったばかり）
- 行動のハードル自体が最大の障壁になっている層（始める前に止まってしまう人）
- 「無料なら見てみる」「特典があるなら登録する」で動く、損得に素直な層

逆に、半信半疑で証明を求める層や、自己否定で動けない層には弱い。そこは型3・型2の領域。

## 書き方の指針

- FVで報酬を出し切る。何が・いくつ・いくらで手に入るかを、もったいぶらずに冒頭で見せる。
- 「できない理由はもう無い」を作る。お金・スキル・時間・リスクの4つの障壁を、それぞれ「かからない/要らない/短い/損しない」で潰す。
- CTAごとに特典と「無料」を再掲する。読者は前半を読み飛ばすので、ボタンの近くに必ず報酬を置き直す。
- 悩みパートは軽くてよい。深掘りせず「だからこれが要る」へ素早く渡す。証明・提供者・安心条件はYESを後押しする補強として置く。
- 数字は入力にあるものだけ使う。人数・満足度・実績を創作しない。無いなら空欄プレースホルダ `[ ]` のまま残す。
- `references/japanese-expression-rules.md` の禁止表現・常套句に従う。

## セクション構成と雛形

```md
# [オファー名／タイトル：手に入るものが一目で分かる短い言葉]

## FV（報酬先出し）

[誰のための、何が無料で手に入るのか]を1行で。
例の構造: 「[対象]が、[理想の状態]になるための[提供物]を、[無料／0円／特典付き]で。」

- [提供物その1：受け取れる中身]
- [提供物その2]
- [提供物その3]

[ 実績数値があればここに1つだけ：登録[数値]／利用[数値]／継続率[数値]。無ければこの行は消す ]

### CTA
今すぐ[受け取る／登録する／予約する]
└ [提供物の要約]を[0円／無料]で。[受け取りにかかる手間：例 30秒で完了／メール登録だけ]。

## 特典・中身（提供価値）

- [特典1：名称]｜[ひと言で何が得られるか]
- [特典2：名称]｜[ひと言]
- [特典3：名称]｜[ひと言]
- [特典4：名称]｜[ひと言]

[ 通常[価格]相当の内容、という根拠が入力にあれば1行。無ければ書かない ]

### できない理由を、ここで消す

- お金：[0円／無料]。[追加費用の有無を正直に]。
- スキル：[未経験／知識ゼロでも進められる理由：例 順番どおりに進めるだけ]。
- 時間：[1回[所要時間]／[期間]で受け取れる]。
- リスク：[いつでも解除できる／登録解除は[手順]で完了]。

### CTA
[受け取る／登録する]
└ [特典の要約]を[無料]で。[手間の軽さ]。
```

## この型を出す前のチェックリスト

- [ ] FVで「何が・いくつ・いくらで（＝無料）」手に入るかを言い切ったか。もったいぶっていないか。
- [ ] 「できない理由」をお金・スキル・時間・リスクの4軸で潰したか。
- [ ] CTAごとに特典と「無料／0円」を再掲したか（前半を読み飛ばされる前提）。
- [ ] 悩みパートを深掘りしすぎていないか（この型では軽く、すぐ報酬へ渡す）。
- [ ] 実績数値・人数・声・受賞を、入力に無いのに作っていないか。無いなら空欄のまま残したか。
- [ ] 希少性（締切・人数）に根拠があるか。根拠が無いのに限定を書いていないか。
- [ ] 解除・費用・取り扱いなど、受け取りの不安になる点を安心条件で先に明かしたか。
- [ ] 禁止表現・LP常套句を使っていないか（`references/japanese-expression-rules.md`）。太字を使っていないか。
