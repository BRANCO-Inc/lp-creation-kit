# LP原稿監査チェックリスト（99点ルーブリック）

下書き後、この基準で採点する。99点未満なら、監査結果を出さずに本文を直してから最終出力する。型名は型1 / 型2 / 型3 で表記する。

## 採点の前提（fail-close ゲート）

以下が1つでもあれば、点数に関わらず即不合格。直すまで出さない。

- [ ] 入力にない実績・人数・満足度・成功率・No.1・受賞・メディア掲載を創作している
- [ ] em dash（—）・全角ダッシュが残っている
- [ ] 「これにより」が残っている
- [ ] 誘導の最終行動が複数混在している（誘導先は登録/予約/受け取りなどでよいが、入力で決めた1つの行動に統一されているか。別行動の説明が混ざっていないか）
- [ ] 担当した型のセクション順・心理から外れている

## 配点（100点満点）

### 1. 入力整合（15点）
- ターゲットの属性がFVか悩みに出ている（3）
- オファー名（タイトル）がFVとCTA付近にある（3）
- コンセプト（できる理由）が型の反転か解決策に反映されている（3）
- 特典・無料・所要時間・日程が入力と矛盾していない（3）
- 数字はすべて入力にある根拠のみ（3）

### 2. 構成・型忠実度（15点）
- 担当した型のセクション順に従っている（5）
- FVが型の心理（接近欲求／自己投影／信頼転移）に合っている（5）
- CTAが正しい位置に置かれている（5）。回数は3回で満点（FV直下・中盤・クロージングが基本）。多すぎてしつこい方を減点する。3回だからと減点しない

### 3. フック強度（15点）
- FVの第一行で、ターゲットの手が止まる強さがある（6）
- 一般論ではなく、具体的な場面・数字・心の声で入っている（5）
- サブコピーで「誰が・何を・どう受け取れるか」が伝わる（4）

### 4. コピー品質＝巧さ（25点）★最重要
`japanese-expression-rules.md` の「巧い表現の作り方」がどれだけ使えているか。

- 抽象語を具体的な数字・場面に置き換えている（5）
- 口語・人間味で広告臭が消えている（4）
- 反転で自己否定を解除している（4）
- プリバンキング or 通説の肯定→反転がある（型2 / 型3で特に）（3）
- 損失回避に寄せ、押し付けになっていない（3）
- クロージングが余韻 or 二者択一で締まっている（3）
- 利用者の声が具体的な行動＋変化＋一人称でリアル（3）

### 5. LP臭の排除（20点）★最重要
- 禁止表現ゼロ（重要です／効果的です／活用する／最適な 等）（5）
- LP常套句なし（「豪華」「完全無料」「最短最速」「新時代の」「市場価値の高い」「ワクワクする未来」）（5）
- 「あなたも◯◯できる！」の連呼、擬態語連発がない（4）
- 太字（`**`）で強調していない（見出し分割と改行で表現）（3）
- 同じ意味の言い換えを繰り返していない（3）

### 6. 誘導・日本語（10点）
- CTAボタンが登録/予約/受け取りの行動を明示している（2）
- 受け取った後に何が届くか分かる（2）
- 見出しが状態／心の声の描写になっている（2）
- 文末が均一でない（断定・体言止め・口語が混ざる）（2）
- H2 / H3が短く、1セクション詰め込みすぎていない（2）

## 99点の壁（95点と99点を分けるもの）

配点で95点まで行っても、ここが弱いと99点には届かない。最後にこの観点で読み返す。

1. FVを声に出して読んで、3秒で「続きを読みたい」と思うか。思わなければFVを書き直す
2. どこか1箇所でも「いかにもLPっぽい」行があるか。1行でもあれば、その行を具体・口語・余韻で書き換える
3. 利用者の声が、台本に見えないか。「（笑）」「2年早く知りたかった、、、」のような崩しが1つでも入っているか
4. クロージングが、煽りでなく余韻で終わっているか。「あなたも変われる！」で終わっていたら書き換える
5. その型でしか書けない原稿になっているか（型1 / 型2 / 型3を入れ替えても成立する没個性な原稿になっていないか）

5項目すべてに自信を持ってYesと言えて、初めて99点。

## 修正手順

1. fail-close ゲートを全消し
2. 根拠のない数字・実績を削る
3. LP常套句・禁止表現を検索して書き換える（具体・口語・余韻へ）
4. FVと利用者の声を「99点の壁」基準で磨く
5. 太字を消し、見出し分割と改行で強調をつけ直す
6. 最終出力はLP本文Markdownだけに整える
