# 誘導LPライティング

## Overview

「誘導LP」の本文Markdownを作る。誘導LPとは、商品本体をその場で売るのではなく、無料オファー・体験・予約・診断・登録など次の一歩へ読者を運ぶためのLP。CTAの最終行動は購入ではなく登録や予約になる。

無料オファーで誘導する例:

- 無料の勉強会・説明会・体験会への参加
- 無料診断・無料相談・無料カウンセリングの予約
- 資料・テンプレート・動画など特典の受け取り
- メッセージ登録（LINE・メール等）

このSkillの中核は3つ。

1. 3段階ヒアリング: ターゲット → コンセプト → その他情報の順で、ユーザーへの質問と一次情報ファイルの2系統で材料を集める
2. 3型を並列生成: 集めた材料から、性格の異なる3つの型を各1エージェントで同時に書き起こす
3. 99点まで自己監査: 各原稿を99点ルーブリックで採点し、未達なら直してから出す

このSkillは単体で完結する。材料はユーザー入力・添付ファイル・この `references/` だけで足りる。外部の非公開ナレッジに依存しない。

## いつ何を読むか

参照ファイルはすべてこのSkillの `references/` 配下にある。相対パスで開く。

| タイミング | 読むファイル |
|---|---|
| ヒアリングを始める前 | `references/hearing-guide.md` |
| 型を割り振る前 | `references/templates/` の3ファイル |
| 各エージェントが書く時 | 担当テンプレ ＋ `references/line-lp-structure.md` ＋ `references/japanese-expression-rules.md` |
| 書いた後の採点 | `references/audit-checklist.md` |

## ワークフロー

### STEP 0. ヒアリング（3段階）

`references/hearing-guide.md` を開き、その順番で進める。要点だけ再掲する。

1. ターゲット: ①ユーザーへの質問（悩み・理想を思いつくまま話してもらう）＋②ターゲットの一次情報ファイル（顧客面談・アンケート・既存顧客の声など）のパス/URL
2. コンセプト: ①ユーザーへの質問（一言で言うと/タイトル/中身/できる理由/信じてほしいこと）＋②コンセプトの一次情報ファイル（資料・台本・商品設計メモ）
3. その他情報: ①ユーザーへの質問（特典・実績・提供者・オファー・CTA・安心条件・緊急性）＋②概要をまとめたファイルのパス/URL

ルール:

- 各段階で「ユーザーへの質問」と「一次情報ファイルの依頼」を必ずセットで出す。質問は段階ごとに区切り、一度に全部投げない
- 一次情報が渡されたら読む。ローカルファイル・URL・クラウド上のファイルのいずれも、その場で読んで生の言い回しを拾う
- ユーザーが「全部入力済み」「もう材料はある」と言ったら、不足項目だけ確認して先へ進む
- 音声入力で材料が来る前提。フィラー・言い直し・重複は無視して意図を抽出する

### STEP 1. 材料を1枚のブリーフに圧縮する

3段階で集めた材料を、生成エージェントへ渡す共通ブリーフに圧縮する。長い一次情報も必ずこの形へ要約する。ジャンル中立に書く（教育・美容・金融・医療・サービス、どれでも同じ枠で埋まる）。

```md
## LPブリーフ
ターゲット:（属性 / 今の行動 / 刺さる悩み / 避けたい未来 / なりたい状態）
刺さる悩みTOP3:（具体的な生活場面の言葉で。3つ）
信じさせたいコンセプト:（一言で / できる理由 / 既存の選択肢との違い）
オファー名:（無料オファー・体験・診断・登録の名称。仮でも可）
オファー条件:（無料/有料 / 所要時間 / 形式 / 日程や開催方法）
特典:（数 と 各特典名。無ければ「なし」）
実績・数値:（入力にある根拠のみ。創作禁止）
提供者:（名前 / 肩書き / 信じてもらえる理由）
安心条件:（初心者OK・オンラインOK・顔出し不要・勧誘なし など）
緊急性:（人数制限・締切。根拠がある時だけ）
CTA最終行動:（登録 / 予約 / 特典受け取り のどれか1つ）
```

CTA文言やURLが未確定なら、本文では仮置きで `無料で受け取る` を使う。

### STEP 2. 3型を並列生成する（3エージェント / 1型1エージェント）

3つの型を、それぞれ別エージェントで同時に書く。1型につき1エージェントを並列でスポーンする。

| エージェント | 担当テンプレ | 型 |
|---|---|---|
| Agent 1 | `references/templates/type-1-offer.md` | 型1 オファー先出し型（報酬を先に見せる） |
| Agent 2 | `references/templates/type-2-mirror.md` | 型2 自己投影・反転型（読者自身を鏡に映す） |
| Agent 3 | `references/templates/type-3-proof.md` | 型3 権威・実証先出し型（信頼を先に立てる） |

各エージェントへ渡す指示（このテンプレを3つ作り、担当テンプレ名だけ差し替えて同時スポーン）:

```
誘導LPの本文をMarkdownで書くタスクです。

まず以下を順に読む（すべて相対パスでこのSkill内にある）:
1. references/templates/{担当テンプレ}
2. references/line-lp-structure.md
3. references/japanese-expression-rules.md

LPブリーフ:
（STEP1のブリーフを貼る）

担当テンプレの型・セクション順・起点心理に厳密に従って本文Markdownを書く。
- japanese-expression-rules.md の禁止表現はゼロ許容。書き終えたら検索して消す
- 入力にない実績・人数・満足度・受賞・掲載は創作しない。根拠のある数字だけ使う
- 説明や制作意図は本文に混ぜない。LP本文だけを出力する
- 出力後、references/audit-checklist.md で自己採点し、99点未満なら直してから出す

出力: LP本文Markdown（先頭に型名のコメントを1行だけ付ける）
```

3型を割り振る理由: 同じ材料でも、報酬で動く層・共感で動く層・信頼で動く層は手が止まる入口が違う。3本を並べて一番効く入口を選べるようにする。

ユーザーが「型2だけ」「1本でいい」と指定したら、その型だけ生成・出力する。

### STEP 3. 統合・99点自己監査・提示

3本が返ってきたら、メインが `references/audit-checklist.md` で各本を再採点する。

- 99点未満の本は、該当エージェントへ差し戻すか、メインが該当箇所だけ直す
- 3本に共通の弱点（同じ禁止表現・同じ根拠なし数字）があれば、テンプレ/ルール側の不足としてメモする
- 3本のうち、ブリーフのターゲットに最も合う1本を「推奨」として明示する

最終出力:

```md
## 推奨: 型2 自己投影・反転型
理由:（ターゲットが「やる気はあるが挫折経験あり」のため、自己否定を解除する入口が最も効く）

---
（型1 全文）
---
（型2 全文）
---
（型3 全文）
```

## 出力方針

- 最終出力はLP本文Markdownを主役にする
- セクション名は `references/line-lp-structure.md` の標準セクション名（FV / 実績バッジ / CTA / 悩み / 問題提起 / 反転 / 解決策 / 中身 / 特典 / 実証 / 提供者プロフィール / 安心条件 / 受け取りSTEP / FAQ / クロージング）に揃える
- デザイナー/実装者がそのまま画像LPに載せられる粒度で書く（H2/H3は短く、1セクションに詰め込みすぎない）
